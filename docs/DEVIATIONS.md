| Match Timeline AND Lineups sourced from TxLINE, not API-Football (`getMatchTimeline` + `getMatchLineups` in `src/server/txline/adapter.ts`; `TimelineEvent`/`MatchLineups`/`TeamLineup`/`LineupPlayer` in `txline/types.ts`; both `/api/matches/[id]/{events,lineups}` rewired) | API-Football does not cover TxLINE's simulated World Cup fixtures (lookup by team-name + date returns no match), and its key was invalid — but TxLINE's own scores feed turned out to carry far more than first assumed. Timeline: goals from `Score.Total.Goals` deltas, yellow/red from `Stats` keys 3/4/5/6, substitutions from `substitution` action rows (deduped by `Id`), team from `Participant1IsHome`. No scorer names (goal/card rows carry no player id), so the Timeline UI (centre-spine layout) renders team + minute + running score. Lineups: parsed from the `lineups` action record — real player names, shirt numbers, starter/bench, position buckets (positionId 34/35/36/37 → G/D/M/F), formation derived from starter counts. Availability depends on fixture coverage (`CoverageType: "TV/Stream"`); uncovered fixtures return null → the Lineups UI shows a formation preview. `src/server/football/adapter.ts` and `API_FOOTBALL_KEY` are now fully unused (kept in-tree, not deleted). | 2026-07-15 |
| Match/checkin/events/lineups now fall back to the live TxLINE feed when a fixture isn't in the DB | The DB (`matches`) is only seeded by the worker's fixture sync, so any fixture visible on the home page but not yet synced (every friendly, or a World Cup fixture before the worker's first run) previously redirected home / 404'd, and `checkins.match_id`'s FK to `matches` made check-in fail outright. The match page, `/api/checkin` (which now upserts the match row first), and both `/api/matches/[id]/{events,lineups}` routes resolve from `listWorldCupMatches()` when the DB misses. | 2026-07-15 |
| `INCLUDE_FRIENDLIES` env flag (`src/server/txline/adapter.ts`, `.env.example`) | Off by default to stay World Cup-only (NG-5).
